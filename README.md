## 📖 О проекте
`quiz_project` — это Python-приложение, представляющее собой интерактивную викторину, запускаемую в терминале. Проект был разработан по заданию предприятия и демонстрирует широкий спектр техник разработки: от работы с данными в формате JSON до модульного тестирования и разделения логики.


---
## 📂 Структура проекта
```
quiz_project/
│
├── data/                  # Модуль данных: вопросы, настройки
│   └── questions.json      # Файл с вопросами в формате JSON
│
├── business_logic/        # Бизнес-логика игры (ядро)
│   ├── __init__.py         # Инициализация пакета
│   ├── question_generator.py  # Загрузка и выдача вопросов
│   ├── answer_processor.py     # Проверка ответов пользователя
│   └── score_manager.py        # Управление подсчетом очков
│
├── interface/             # Интерфейс взаимодействия с пользователем
│   ├── __init__.py         # Инициализация пакета
│   └── console_ui.py       # Консольный интерфейс для терминала
│
├── tests/                 # Тесты для модулей бизнес-логики
│   ├── test_question_generator.py
│   ├── test_answer_processor.py
│   └── test_score_manager.py
│
├── main.py                # Точка входа в приложение
└── requirements.txt       # Список зависимостей проекта (например, для pytest)
```

## ⚙️ Примеры работы
**Запуск викторины:**
При запуске программы из корневой папки проекта пользователь попадает в консольный интерфейс, где ему предлагается ответить на вопросы.

```bash
$ python main.py

## 🎉 Добро пожаловать в Викторину!

**Тема: Общие знания**
Вопрос 1: Столица Франции?
1) Берлин 2) Лондон 3) Париж 4) Рим
Ваш ответ: 3
✅ Правильно!

Тема: История
Вопрос 2: В каком году был основан Санкт-Петербург?
1) 1603 2) 1703 3) 1803 4) 1903
Ваш ответ: 2
✅ Правильно!
...

🏁 Игра окончена!
Ваш итоговый счет: 2 из 2 (100%)
```
**Изменение вопросов:**
Вы можете легко изменить вопросы или добавить новые, отредактировав файл data/questions.json в любом текстовом редакторе.

## 💡 Использование
**Шаг 1: Установка зависимостей**
Установите необходимые библиотеки из файла requirements.txt. В зависимости от содержимого файла, это могут быть библиотеки для тестирования (например, pytest) или другие зависимости.

``` bash
pip install -r requirements.txt
```

**Шаг 2: Запуск программы**
Запустите викторину из терминала, находясь в корневой папке проекта.

``` bash
Копировать
python main.py
```

**Шаг 3: Запуск тестов (для разработчиков)**
Для проверки корректности работы бизнес-логики можно запустить модульные тесты.

```bash

pytest tests/
```

## 🧠 Логика анализа

Алгоритм работы приложения:
1. Инициализация: main.py запускает консольный интерфейс.
2. Загрузка данных: Модуль question_generator.py считывает вопросы из data/questions.json.
3. Цикл игры: console_ui.py выводит вопрос на экран и получает ответ от пользователя.
4. Проверка: Ответ пользователя передается в answer_processor.py для проверки на корректность.
5. Подсчет очков: В зависимости от результата проверки, score_manager.py обновляет текущий счет игрока.
6. Завершение: После ответа на все вопросы интерфейс выводит финальный счет и завершает игру.
7. Тестирование: Модули в директории tests/ используют фреймворк (например, pytest) для изолированной проверки функций из business_logic.

## 🛠️ Технологии
- **Python 3.8+**
- JSON: Формат для хранения и обмена данными (вопросы викторины).
- Pytest (или другой фреймворк): Инструмент для написания и запуска модульных тестов.
- Встроенные модули Python: Для работы с файловой системой (json, os), логикой и взаимодействия с пользователем.
